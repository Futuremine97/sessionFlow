# Session Transfer

Move a conversation's working context (decisions, open threads, user
preferences, artifacts) from one AI session to another. The output is a
**primer** the user pastes as the first message of the new session.

## When to use
- "Hand this off to Claude / ChatGPT / Gemini"
- "I'm hitting the context limit — continue in a fresh chat"
- "Switch this to a better model without losing context"
- User pastes a conversation export and wants to resume elsewhere

## How to run it

This plugin bundles a dependency-free Node CLI at
`${CLAUDE_PLUGIN_ROOT}/scripts/s2s.js` (requires Node 18+).

```bash
# From a pasted conversation (file or stdin):
node "${CLAUDE_PLUGIN_ROOT}/scripts/s2s.js" paste path/to/chat.txt --to claude

# From a vendor export (auto-detects claude/chatgpt/gemini):
node "${CLAUDE_PLUGIN_ROOT}/scripts/s2s.js" transfer export.json --to gemini --mask

# Merge several sessions into one project memory:
node "${CLAUDE_PLUGIN_ROOT}/scripts/s2s.js" merge a.json b.json --to claude

# See what changed between two capsules:
node "${CLAUDE_PLUGIN_ROOT}/scripts/s2s.js" diff old.capsule.json new.capsule.json
```

Useful flags: `--to <target>`, `--from <source>` (else auto-detect),
`--full` (include verbatim transcript), `--mask` (redact secrets/PII),
`--offline` (force heuristic summary), `-o file`, `--capsule file`.
If `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GEMINI_API_KEY` is set, the
summary is generated by that LLM; otherwise a heuristic is used.

## Steps
1. Identify the source conversation (a file the user points to, an export, or
   text they pasted) and the **target** platform they're moving to. If the
   target is unclear, ask once; default to `claude`.
2. Run the bundled CLI with the right subcommand (`paste` for raw text,
   `transfer` for exports). Keep `--mask` on unless the user opts out.
3. Show the returned **primer** in a copyable block and note in one line what
   was carried over (e.g. "3 decisions, 2 open threads, 1 file"). Don't dump
   the whole capsule unless asked.

## If Node isn't available
Fall back to doing it by hand: read the conversation, then produce a primer
with these sections — Origin, Summary, User profile & preferences, Key facts,
Decisions made, Artifacts/files, Open threads (pick up here), Glossary. For a
Claude target wrap it in `<handoff>...</handoff>`; for ChatGPT/Gemini use plain
markdown headers. Redact any API keys, emails, or secrets first.
